# 飲食店向け 原価計算アプリ 仕様書（初版）

## アプリ概要
飲食店におけるメニューの原価計算および原価率の管理をモバイル端末（iOS/Android/Mac/PC）のブラウザ上でオフラインでも行えるPWA（Progressive Web App）アプリケーション。データは端末内に保存し、完全無料で運用可能な構成とする。

## ターゲットユーザー
飲食店のオーナー、店長、料理長など、メニューの価格設定や原価管理を行う担当者。

## 主な機能要件

### 1. 食材管理機能（Ingredient Management）
使用する食材の基本情報を登録し、一覧で管理する機能。

*   **データ項目:**
    *   食材名（例：玉ねぎ、牛ひき肉、醤油）
    *   仕入価格（円）
    *   仕入容量（単位付き。例：1000g、1本、500ml など）
    *   単位当たりの単価（自動計算：仕入価格 ÷ 仕入容量）
*   **機能:**
    *   新規食材の登録（連続して登録しやすいUI）
    *   登録済み食材の一覧表示・検索
    *   食材情報の編集・削除
    *   **※重要:** 食材の「仕入価格」や「容量」が変更された場合、その食材を使用しているすべてのメニューの原価が自動的に再計算される。

### 2. 原価管理（メニュー管理）機能（Cost Management）
登録された食材を組み合わせてメニューを作成し、原価と原価率を把握する機能。

*   **データ項目:**
    *   メニュー名（例：特製ハンバーグ定食）
    *   メニュー画像（任意）
    *   使用食材リスト（食材管理から選択）とそれぞれの使用量
    *   原価合計（自動計算：各食材の「単位当たり単価 × 使用量」の合計）
    *   販売価格（売価設定）
    *   原価率（自動計算：原価合計 ÷ 販売価格 × 100）
*   **機能:**
    *   新規メニューの作成
    *   メニューに対する食材の追加・削除・使用量の設定
    *   原価、販売価格、原価率のリアルタイム計算と表示
    *   メニューの一覧表示
    *   メニューの編集・削除

## 非機能要件
*   **プラットフォーム:** PWA (Progressive Web App)。ブラウザで動作し、ホーム画面に追加可能。Windows環境で開発・テスト可能。
*   **データ保存先:** クライアントサイド（ブラウザ内のローカルデータベース：IndexedDBなど）。サーバー通信を行わずオフラインで動作可能。
*   **デザイン:** モバイル操作を前提としつつ、シンプルでおしゃれなUI（Tailwind CSSなどを想定）。**表示言語はすべて日本語とする。**

## 画面構成案（UIイメージ）

基本的な画面構成は以下の3つのエリアで構成します。

1. **左側サイドメニュー:** 主要な機能（食材管理、原価管理、設定など）の切り替え。
2. **上部サブナビゲーション:** メイン画面の上部に配置し、現在表示している機能のカテゴリ分類表示（タブなど）として使用。
3. **メイン画面:** リスト表示やコンテンツの入力などを広く行うエリア。

## 詳細要件・取り決め事項

*   **歩留まり計算:** 実装しない（シンプルさを優先）
*   **人件費:** 原価計算に含めない
*   **包材など副資材:** 「食材管理」の項目の一部として登録し、原価計算に含めることが可能
*   **単位の取り扱い:** `g`, `ml`, `個` の3種類に限定して運用する（「本」や「尾」などはすべて「個」として入力）
